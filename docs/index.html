Rebound

A TUI tool to set up 0xClumzZy's Arch Linux environment on any Arch-based PC

Rust TUI Arch Linux MIT License
Get Started View on GitHub

✦ Features

⌨

Interactive TUI

Dark Catppuccin-inspired terminal interface with intuitive navigation using arrow keys, spacebar, and keyboard shortcuts.

⚡

Auto Mode

Non-interactive --auto flag selects all tasks and runs immediately. Perfect for scripting and automated deployments.

📦

8 Setup Categories

Granular task selection across Base Packages, AUR Helper, BlackArch, Dev Tools, Shell Config, Desktop, Services, and Wallpapers.

📁

Embedded Dotfiles

All configuration files are embedded directly in the binary. No external dotfile repo needed - everything deploys automatically.

🖼

34 Wallpapers

Dark aesthetic wallpapers included and deployed to ~/Pictures/wallpapers/ with a single task selection.

🔒

Secure Password Input

Sudo password prompted before TUI starts with hidden input (displayed as stars). Cached for passwordless installs.

📊

Live Progress

Real-time progress bar with animated spinner, colored output log, and task completion tracking.

🎨

Dark Theme

Deep dark palette with high-contrast accents. Easy on the eyes during extended setup sessions.

📥 Installation

Requirements: Arch Linux (or Arch-based distro like EndeavourOS, Garuda, Manjaro), Internet connection, ~500MB disk space.

One-liner (Recommended)

bash 
curl -sL https://raw.githubusercontent.com/0xClumzzy/rebound-/main/install.sh | bash

Direct Binary Download

bash 
curl -sL https://github.com/0xClumzzy/rebound-/releases/download/v1.0.0/rebound-x86_64 -o ~/.local/bin/rebound
chmod +x ~/.local/bin/rebound

Build from Source

bash 
git clone https://github.com/0xClumzzy/rebound-.git
cd rebound-
cargo build --release
cp target/release/rebound ~/.local/bin/

PATH Setup

Ensure ~/.local/bin is in your PATH:

bash 
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc

🚀 Usage

rebound Interactive TUI mode
rebound --auto Install everything automatically (no TUI)
rebound -a Same as --auto
rebound --help Show help message
rebound -h Same as --help

Interactive Mode Workflow

  1. Launch rebound
  2. Enter sudo password when prompted (shown as ***)
  3. Browse categories with ↑ ↓ arrow keys
  4. Press Enter to view tasks in a category
  5. Select/deselect tasks with Space
  6. Press a to toggle all tasks in current category
  7. Press Backspace to clear all tasks in category
  8. Press Enter to start installation
  9. Watch live progress with output log
  10. Press q or Esc to go back or quit

Application States

rust 
enum AppState {
    Splash,          // Initial splash screen (1.5s auto-advance)
    SelectCategory,  // Browse & select categories
    SelectTasks,     // Toggle individual tasks within a category
    Running,         // Tasks executing in background thread
    Done,            // All tasks finished, show summary
}

Data Flow

  1. Startup: main() parses args, prompts sudo password, initializes terminal
  2. Splash: Animated logo displayed for 1.5 seconds
  3. Category Selection: User browses 8 categories
  4. Task Selection: User toggles tasks within chosen category
  5. Execution: Runner spawns background thread, executes shell commands sequentially
  6. Progress: Shared Arc<Mutex<T>> state updated in real-time, UI polls every 50ms
  7. Completion: Success/failure summary displayed

Runner Implementation

The Runner struct uses thread-safe shared state for concurrent UI updates:

rust 
pub struct Runner {
    pub state: Arc<Mutex<RunState>>,        // Idle | Running | Done
    pub output: Arc<Mutex<Vec<String>>>,   // Log lines
    pub current: Arc<Mutex<usize>>,        // Progress counter
    pub total: Arc<Mutex<usize>>,          // Total steps
    pub current_name: Arc<Mutex<String>>,  // Current task name
    pub success: Arc<Mutex<bool>>,         // All succeeded?
    pub tick: Arc<Mutex<u64>>,             // Animation frame
    pub failed_tasks: Arc<Mutex<Vec<String>>>,
    pub last_progress: Arc<Mutex<Instant>>,
}

🔧 Troubleshooting

Sudo password not accepted

Rebound prompts for your sudo password before the TUI starts. If you enter it incorrectly, you'll see a warning but the tool continues. Some tasks may fail without sudo access.

Fix: Run sudo -v in another terminal to cache your password, then restart rebound.

Command not found: rebound

The binary isn't in your PATH.

Fix: Ensure ~/.local/bin is in your PATH or run with full path: ~/.local/bin/rebound

Pacman lock error

Another pacman process is running (possibly a system update or package manager GUI).

Fix: Wait for the other process to finish, or check with pgrep pacman.

BlackArch strap.sh fails

Network issues or the BlackArch mirror is temporarily down.

Fix: Try again later or manually download strap.sh from https://blackarch.org/strap.sh.

AUR package build fails

Missing build dependencies or AUR package updated.

Fix: Install base-devel first (from Base Packages category), then retry AUR tasks.

Wallpapers not deployed

Permission denied writing to ~/Pictures/wallpapers/.

Fix: Ensure ~/Pictures exists and is writable, or run with sudo.

❓ FAQ

What Linux distros are supported?

Arch Linux and Arch-based distributions including EndeavourOS, Garuda, and Manjaro. The tool relies on pacman as the package manager.

Can I use rebound on non-Arch systems?

No. Rebound specifically uses pacman commands and Arch-specific repositories. It will not work on Debian, Fedora, or other distributions.

How much disk space does a full install use?

Approximately 500MB for all packages plus wallpaper images. The binary itself is small (a few MB) since wallpapers and dotfiles are embedded.

Is it safe to run in auto mode?

Auto mode selects and installs everything. It's designed for fresh installs. On an existing system, some packages may already be installed (they'll be skipped with --needed).

Can I customize the dotfiles?

Yes. Edit the files in src/dotfiles/ and rebuild with cargo build --release. The dotfiles are embedded via include_str! macros.

How do I add my own wallpapers?

Add image files to src/wallpapers/ and register them in src/wallpapers.rs using include_bytes!. The wallpapers are deployed to ~/Pictures/wallpapers/.

Can I add new categories or tasks?

Yes. Add new task functions in src/tasks.rs following the existing pattern. Each category returns a Category struct with a list of Task structs containing shell commands.

Does rebound work with systemd-based Arch spins?

Yes. All service management uses systemctl which is standard on systemd-based Arch systems.

🤝 Contributing

Development Setup

bash 
git clone https://github.com/0xClumzzy/rebound-.git
cd rebound-
cargo build
cargo run

Adding a New Task

  1. Open src/tasks.rs
  2. Add a new Task::new() to the appropriate category function
  3. Provide name, description, and shell commands as vec!["command1", "command2"]
  4. For file deployments, use .with_files(vec![("~/.config/file", content())])
  5. Build and test with cargo run

Adding a New Category

  1. Create a new function in src/tasks.rs returning Category
  2. Add it to the all_categories() vector
  3. The TUI will automatically include it

Code Style

  • Follow existing patterns in the codebase
  • Use the theme helpers from theme.rs for consistent styling
  • Keep commands idempotent where possible (use --needed flag)
  • Test on a fresh Arch install or VM before submitting
